# /briefing

Generate a personalized 6-8 page branded PDF that applies Elixirr's AI transformation frameworks to a specific prospect's industry, company, and role.

## Usage

```
/briefing John Smith, CTO, Acme Corp
```

Or just `/briefing` and provide details when prompted.

---

## How It Works

```
┌─────────────────────────────────────────────────────────────────┐
│                    PROSPECT BRIEFING                             │
├─────────────────────────────────────────────────────────────────┤
│  1. Collect prospect details (name, title, company)             │
│  2. Research prospect, company, and industry + AI context       │
│  3. Apply frameworks: Three Forces, Maturity Model,             │
│     Failure Patterns, Augmented Professional, Outcome-First     │
│  4. Draft as Markdown for your review                           │
│  5. Iterate until approved                                      │
│  6. Generate branded Elixirr PDF                                │
└─────────────────────────────────────────────────────────────────┘
```

---

## What I Need From You

- **Name** — Full name of the prospect
- **Title** — Their role (e.g., "Chief Technology Officer")
- **Company** — Full company name
- **Notes** (optional) — How they showed interest, what they care about, relationship context

---

## What You Get

A branded PDF with five sections:

1. **The Forces Reshaping [Industry]** — Three Forces applied to their world
2. **What This Means for [Company]** — Maturity placement + failure patterns to watch for
3. **Leading Through the Shift** — Role-specific implications + Augmented [Role] table
4. **Where I'd Focus First** — 3 sequenced priorities with diagnostic questions
5. **A Conversation Worth Having** — Soft CTA referencing the full Operating System

The draft comes as Markdown first so you can review and revise before generating the final PDF.
